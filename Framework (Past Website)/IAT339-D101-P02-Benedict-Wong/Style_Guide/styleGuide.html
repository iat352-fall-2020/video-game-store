Interactive Text

In-text Link

In-text links are used in news articles mainly when pages, either internal or external, can guide the user to a separate site for more information. This, for example, links back to this section. The HTML code for an in-text link is shown below.

HTML code
<p>This, for example <a href="#inter-text">links</a>back to this section.</p>
CSS code

At Footy Fans, we also use a special CSS styling for anchor tags (which are used for the links). This special styling removes the underline style in links, as well as the purple color when the link has been visited. The latter is done to have link colors stay consistent at all times. The code below shows the styling that is used on the anchor tags.


            a{
              text-decoration: none;
              padding-right: 1em;
            }
            a:visited{
              color:blue;
            }

Text Input Field and Label

Text input is used throughout the Footy Fans website, from entering information into a form to writing a review for a product. The text input area allows the user to input text, which can be turned into meaningful data. The code below shows how this can be implemented in HTML.

HTML code
<input type="text" name="text-input" id="first-name">
CSS code

The text input can also be resized so that the text input is less claustrophobic. Using CSS, we can add padding to the text input so that there is more space for the text area. A padding of 2em for the top and bottom and 5 em for the left and right of the text input area is shown below. 


              input[type=text]{
                padding: 0em 2em;
              }

The label on the other hand helps put a label on a input field. This helps the user to identify what an input field is used for. This also links the label with the input element using the for and id attributes for the label and input, respectively. For example, in the purchase screen, the label is used to prompt the user to enter their first name, their last name, etc. The code below shows how this is implemented.

HTML code
<label for="first-name">

Navigation Button Style

At Footy Fans, there are two types of navigation button styles. First, there are the navigation buttons that link to the main content of the website. For this we use rectangular shape buttons that blend in with the top bar. On the other hand, there are the contact buttons that link to social media, as well as contact and addresses. These buttons are located at the footer for the mobile site or the header, along with the other navigation buttons, on the desktop site.

Header Buttons

The buttons in the header, excluding the social media and contact buttons, are composed of the News, the Collections, the Wishlist, the Account and the Cart sections. These buttons blend in with the header bar. First, we define the HTML code for these buttons. The code below is for the News button:

HTML code
<button class="header-nav-button" onclick="linkTo('news.html')">News</button>
CSS code

Next, we style the button using CSS so that it blends in with the header's color and style: 


              .header-nav-button{
                background-color: #020f3c;
                padding: 1em 3em;
                font-family: 'Roboto', sans-serif;
                border: none;
                border-right: 0.3em solid white;
                color:white;
                transition: background 0.3s  ease-out;

              }
JavaScript code

Lastly we add JavaScript code so that the click is meaningful and links to a desired webpage: 


               function linkTo(url){
                 location.href=url;
               }

Text Styling

Fonts Headings/Paragraphs Lists

Fonts

At Footy Fans, we use three sans-serif fonts in our website. These fonts are Anton, Open Sans and Roboto. These three fonts are used in our branding, our headings and our content text, respectively.

Anton represents our branding.

Anton is a sans-serif font with bold block lettering that helps emphasize the importance of our branding. Since it is used only in our branding, Anton will not be used unless it is for the most important titles.

Open Sans represents our headings.

Open Sans is a font that we use on our headings. Unlike Anton or Roboto, Open Sans fits perfectly in the middle between bold lettering and thin lettering used for reading. By being in the middle of bold and thin, Open Sans helps the user to identify that the text with Open Sans represents a heading.

Roboto represents our content text.

Lastly, Roboto is a font that we use in our body text. While bold and medium lettering can help the user to identify importance, they are not the best for reading. Instead, Roboto is a thin font that we use so that our content is readable and comfortable for the user.

Headings and Paragraphs

Headings

At Footy Fans, we use three main sizes of headings to help the user to identify important content, group content together and provide information as to what part of the website the user is currently on.

HTML code

Headings follow a numbered system starting from 1 and ending at 6, each representing a level of importance and font size. The HTML code below shows the heading code that we use for the style guide, showing examples of headings that are used here.

<h1>Footy Fans Style Guide</h1> <h2>Text Styling</h2> <h3>Headings and Paragraphs</h3>
CSS code

Next, as part of Footy Fans' font style, we implement CSS so that the font used for the headings is Open Sans.


              h1, h2, h3{
                font-family: 'Open Sans', sans-serif;
              }

Paragraphs

In HTML, content uses the paragraph tags.

HTML code
<p>This is a rather short paragraph.</p>
CSS code

As part of Footy Fans' font style, we implement CSS such that Roboto is the font used for content.


              p{
                font-family: 'Roboto', sans-serif;
              }

Lists

As Footy Fans is a football apparel website that allows users to browse and purchase merchandise, lists are used throughout the website. For example, lists are used to list out the items that exist at the company or provide rankings for best-selling merchandise.

Numbered List

Numbered lists exist throughout Footy Fans. One example of a numbered list that exists is through the rankings of the best-selling merchandise on Footy Fans. The best-selling merchandise start from 1 and end at 5, showing the best-selling items at the company.

HTML code

A numbered list is known as an ordered list in HTML, in that the list items are ordered starting from first to last. Hence, we use the ol tag to denote that the following list is an 'ordered list'


              <ol>
                <li>Item 1</li>
                <li>Item 2</li>
                <li>Item 3</li>
              </ol>
CSS code

Since a numbered list is part of the content, we must style the list so that it uses our content font, Roboto.


              ol{
                font-family: 'Roboto', sans-serif;
              }

Bulletted List

Unlike Numbered Lists where list items are numbered in order, Bulletted Lists list out each item as a bullet point. This is useful as there are lists where order is unimportant. For example, in the individual product page, information regarding the make and apparel elements uses a bulletted list in order to just list out information about the individual product.

HTML code

A bulletted list is known as an unordered list in HTML, such that a bulletted list does not have any order to it, listing its list items without numbering. We use the 'ul' tag to denote the fact that the list is an unordered list.


              <ul>
                <li>Item 1</li>
                <li>Item 2</li>
                <li>Item 3</li>
              </ul>
CSS code

As part of the Footy Fans font style, we must style this list so that it uses our content font, Roboto.


              ul{
                font-family: 'Roboto', sans-serif;
              }

Main Navigation

The main navigation for Footy Fans is located at the header bar. This bar features most of the pages on the mobile site and all of the pages on the desktop site. This navigation uses flexboxes with three items containers aligned in a single column, like so:

HTML code

In order to achieve this effect, first we must define a flexbox container and the three item containers that will house the content. While the search bar is not part of the navigation, it is part of the header bar, therefore we must create a container that will house the search bar.


              <div class="nav-container box">
                <div class="box-item-logoProfileCart">
                </div>
                <div class="box-item-searchBar">
                </div>
                <div class="box-item-newsCollectionWishlist">
                </div>
              </div>

Next, we will insert the relevant content into each of these respective item containers.


              <div class="box-nav-container">
                <div class="box-item-logoProfileCart">
                  <a href="main.html"><img src="images/logo.png"/></a>
                  <button class="header-square-button" onclick="linkTo('profile.html')">
                  <button class="header-square-button" onclick="linkTo('cart.html')">
                </div>
                <div class="box-item-searchBar">
                  <form>
                    <label for="search-input" id="label-search">Search here: </label><input type="text" name="search-bar" id="search-input">
                    <button type="reset" id="reset-button"><img src="images/reset-icon.png" alt="reset-icon" id="reset-icon" onclick="clearText('search-input')"/></button>
                    <button type="submit" id="search-button"><img src="images/search-icon.png" alt="search-icon" id="search-icon"/></button>
                  </form>
                </div>
                <div class="box-item-newsCollectionWishlist">
                  <button class="header-rectangular-button" onclick="linkTo('news.html')">Header Button</button>
                  <button class="header-rectangular-button" onclick="linkTo('collections.html')">Header Button</button>
                  <button class="header-rectangular-button" onclick="linkTo('wishlist.html')">Header Button</button>
                </div>
              </div>
CSS code

Now that we have implemented the HTML code for the navigation bar, it is time to implement and style the navigation bar with CSS. First we must set the container to use flexbox in a column direction.


              .box-nav-container{
                display: flex;
                flex-direction: column;
                align-items: stretch;
              }

Next, we will set each item container to have use flexbox in a row direction.


              .box-item-logoProfileCart{
                display: flex;
                flex-direction: column;
                align-items: stretch;
              }

Lastly, the first item container features elements that align left and right. Therefore we must style it so that the elements reflect this appearance.


              .header-square-button{
                align-self: flex-end;
              }
JavaScript code

In order to make the buttons have a meaningful click, we set our JavaScript function for linking to other urls.


               function linkTo(url){
                 location.href=url;
               }

Website Grid Structure

At Footy Fans, the website uses three main kinds of grid structure. For the mobile version, we use a single column grid, as a grid with multiple columns will not fit on a mobile screen. On the desktop version, we use both a single column grid, as well as double column grid. The images below show our grid structure for all three of these types:

HTML code

The difference between a single column grid and a double column grid is between the flex items inside the flex container: one item for a single column, two items for a double column. As such, we only need to code the container and the items.

For the single column grid structure, we implement it as such: 


            <div id="box-container">
              <div id="box-item">

              </div>
            </div>

For the single column complex grid structure, we implement it as such:


            <div id="box-container">
              <div id="box-item-container">
                <div id="box-item">

                </div>
                <div id="box-item">

                </div>
              </div>
              <div id="box-item">

              </div>
            </div>

For the double column grid structure, we implement it as such: 


            <div id="box-container">
              <div id="box-item">

              </div>
              <div id="box-item">

              </div>
            </div>
CSS code

As we use flexbox, we must style our CSS code accordingly to each grid structure. For our single column grid structure, we use a flex-direction of column.


              .box-container{
                display: flex;
                flex-direction: column;
                align-items: stretch;
              }

Note that the complex single column uses a nested flexbox. In this example, the first item container uses the flex-direction of row, so we must style the CSS such that it reflects this difference.


              .box-item-container{
                display: flex;
                flex-direction: row;
                align-items: stretch;
              }

For our double column grid structure, we use a flex direction of row so that the items are aligned side by side.


              .box-container{
                display: flex;
                flex-direction: row;
                align-items: stretch;
              }

Product Listing

A product listing on Footy Fans consist of three things: an image of the product, the name and its price. On the mobile page, the image and the name/price are in a row direction, while the image and the name/price are in a column direction for the desktop site:

HTML code

For both the mobile and desktop implementations, we use an unordered list to display our product listings.


              <ul id="box-list-item">
                <a href="item1.html"><li class="product-item">
                  <img src="placeholder1.png" class="product-image" alt="product-image"/>
                  <p>
                  Item1
                  </p>
                  <img src="rating1.png" alt="rating-image"/>
                  <p>
                  Price
                  </p>
                <li></a>
CSS code

First we get rid of the bullet points that are implemented automatically as part of the unordered list.


              #box-list-item{
                list-style: none;
              }

For our mobile version, we style our product listing so that the listing image and information are side-by-side with one another.


              /* This is strictly for the mobile styling. Apply rules accordingly. */
              .product-image{
                float: left;
                margin-right: 0.5em;
              }

For our desktop version however, we style our product listing so that the listing image is above the information.


              /* This is strictly for the desktop styling. Apply rules accordingly. */
              .product-item{
                text-align:center;

              }

Design

Footy Fans Color Swatches

Primary

#253341

rgb(37, 51, 65)

A dark blue color is used as the primary theme for Footy Fans. Not only does the dark blue color represent a professional theme, it is also similar to the main color of the Champion's League, a highly contested and important football competition to many players and fans alike. As the color is professional and important to football fans, we use this color to reflect our professionalism and dedication to football.

Secondary

#952424

rgb(149, 36, 36)

For our secondary color, we use a red color. This color is tertiary as it is not used as often as the primary colors. As red represents a high level of importance, it is imperative that we do not use it often. As a result, this red color is used only for important areas, such as the profile and cart button.

Tertiary

#541D81

rgb(84, 29, 129)

A lighter purple is used as the tertiary color for Footy Fans. This color compliments the darker blue color with its lighter tones and its calming nature so that the page is not too plain.

Background

#D2D5D5

rgb(210, 213, 213)

Lastly, we use a grey/light blue color for our background. Absolute colors such as white can be irritating to a user's eyes, especially when the website is viewed in low light situations. Instead, we use this grey/light blue color as an alternative as it is more comfortable to look at.
